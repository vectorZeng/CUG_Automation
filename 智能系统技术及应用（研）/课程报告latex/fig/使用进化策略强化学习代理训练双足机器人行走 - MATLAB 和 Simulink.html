主要内容

使用进化策略强化学习代理训练双足机器人行走

本示例使用：

  • Reinforcement Learning Toolbox强化学习工具箱
  • Simscape MultibodySimscape 多体

本示例展示了如何利用双延迟深度确定性策略梯度（TD3）强化学习代理训练双足机器人行走。本例中的机器人是在Simscape™ Multibody™中建模的。

相关示例请参见“使用强化学习代理训练双足机器人行走”。有关这些代理的更多信息，请参见双延迟深度确定性（TD3）策略梯度代理。

该智能体通过进化策略强化学习算法（ES-RL）进行训练。ES-RL方法[3]结合了交叉熵法（CEM）与非策略的强化学习算法，如SAC、DDPG或TD3。CEM-RL建立在进化强化学习（ERL）[4]的框架上，标准进化算法选择并进化一个行为体群体，并在过程中生成这些经验。这些经验随后被加入回复缓冲区，用于训练演员和批评者网络。

ES-RL算法的一般流程如下：

  1. 一个演员网络的群体会用随机权重初始化。除了人口外，还会初始化一个额外的actor网络，同时创建一个批评网络。

  2. 随后，参与者群体在与环境的互动过程中被评估。

  3. 演员和批评者通过人口演员评估在填充的数据缓冲区上进行更新。

  4. 所有行为者的适应度是通过与环境的交互来计算的。每集的平均回报作为各自的健身指数。

  5. 选择算符根据总体中存活的行为者根据其相对适应度评分选择。

  6. 幸存的精英演员集合用于生成该演员的下一个群体。

修复随机数流以实现可重复性

示例代码可能涉及在不同阶段计算随机数。在示例代码中各部分开头固定随机数流，每次运行时都能保持该段的随机数序列，并提高结果重现的可能性。更多信息请参见结果可重复性。

用种子和随机数算法Mersenne Twister固定随机数流。有关控制用于随机数生成的种子的更多信息，请参见随机数。0

双足机器人模型

这个例子的强化学习环境是一个双足机器人。训练目标是让机器人以最小的控制力度直线行走。

打开Simulink®模型。

  • Copy openExample Command
    Paste command in MATLAB to download and open example files

  • Open in MATLAB Online
mdl = "rlWalkingBipedRobot";
open_system(mdl)

该机器人采用Simscape™多体™建模。

在该模型中：

  • 在中立0弧度位置，双腿伸直，脚踝平放。

  • 脚接触是通过空间接触力（Simscape Multibody）模块建模的。

  • 该代理可以通过施加有界于之间的关节扭矩，控制机器人双腿上的三个独立关节（脚踝、膝盖和臀部）。实际计算出的动作信号在 之间进行了归一化。[-3,3] N·m-1 and 1

环境为代理提供了以下29项观察。

  • 躯干质心的Y（横向）和Z（垂直）平移。Z方向的平移被归一化到与其他观测值类似的范围。

  • X（前进）、Y（横向）和Z（垂直）平移速度。

  • 躯干的偏航角、俯仰角和滚转角。

  • 偏航、俯仰和滚转影响躯干的角速度。

  • 双腿三个关节（踝关节、膝盖、髋关节）的角度位置和速度。

  • 来自上一个时间步的动作值。

如果发生以下任一情况，发作即告终止。

  • 机器人躯干的质心在Z方向上小于0.1米（机器人下落），在任一方向上都超过1米（机器人向侧移过远）。

  • 滚滚、俯仰或偏航的绝对值都大于0.7854辐射。

以下奖励函数rt，在每个时间步都由[2]启发。

rt=vx−3和2−50ˆ跟2+25Ts真是的−0。02∑我在我t−12

这里：

  • vx是机器人在X方向（向目标方向）的平移速度。

  • 和是机器人从目标直线轨迹的横向平移位移。

  • ˆ跟是机器人质心的归一化垂直平移位移。

  • 在我t−1是前一时间步中关节i的力矩。

  • Ts是环境的采样时间。

  • 真是的是环境的最终模拟时间。

该奖励函数通过为正向前进速度提供正奖励，鼓励智能体前进。它还通过提供持续奖励，鼓励代理人避免发作终止（25Ts真是的）在每个时间步。奖励函数中的其他项是对横向和垂直平移发生重大变化以及过度控制努力的惩罚。

创建TD3特工

本例使用基于进化策略的无梯度优化技术训练TD3强化学习者，以学习双足行走。TD3算法是DDPG的扩展，经过改进，防止Q值被高估，使其更具鲁棒性[3]。

  • 两个批判网络——TD3代理独立学习两个批判网络，并使用最小值函数估计来更新演员（策略）。这样做可以避免通过critic更新中的最大算子高估Q值。

  • 目标策略噪声的添加——在值函数中添加截断噪声可以平滑Q函数值，使类似动作中的Q函数值变得平滑。这样做可以防止学到错误的尖锐峰值噪声估计。

  • 延迟策略和目标更新——对于TD3代理来说，延迟演员网络更新可以让Q函数有更多时间减少错误（接近所需目标），然后再更新策略。这样做可以防止价值估计的差异，从而获得更高质量的策略更新。

指定进化策略训练选项和训练代理

固定随机生成器种子以保证可重复性。

  • Copy openExample Command
    Paste command in MATLAB to download and open example files

  • Open in MATLAB Online
rng(0,"twister");

ES-RL训练需要设置以下几个重要的列车选项。

  • PopulationSize每代评估的演员数量。我们选出20位演员

  • PercentageEliteSize，这代表了成为下一代行为者的父母的幸存精英人口规模。在这个例子中，将精英人数设为总人口的50%。

  • MaxGenerations设定了种群进化的最大世代数。在这个例子中，MaxGenerations 设置为 2000。

  • MaxStepsPerEpisode为每个演员的每集运行设定了最大模拟步数。

  • TrainEpochs设置强化学习代理基于梯度更新的跨时期数值。

  • WeightedMixing将更新方法设置为均值和标准差。

  • 在强化学习训练监视器中显示训练进度（设置选项），命令行显示显示（设置选项）。PlotsVerbose

  • 当客服平均得分达到250时终止培训。

欲了解更多信息及更多选项，请参见rlEvolutionStrategyTrainingOptions。

  • Copy openExample Command
    Paste command in MATLAB to download and open example files

  • Open in MATLAB Online
maxGeneration = 2000;
maxSteps = floor(Tf/Ts);
trainOpts = rlEvolutionStrategyTrainingOptions(...
    "MaxGeneration", maxGeneration, ...
    "MaxStepsPerEpisode", maxSteps, ...
    "ScoreAveragingWindowLength", 10, ...
    "Plots", "none", ... 
    "StopTrainingCriteria", "GenerationReward", ...
    "StopTrainingValue", 250,...
    "PopulationSize",25,...
    "PercentageEliteSize",50,...
    "ReturnedPolicy", 'BestPolicy',...
    "Verbose",1,...
    "SaveAgentCriteria",'none');

trainOpts.TrainEpochs = 50;
trainOpts.EvaluationsPerIndividual = 1;

trainOpts.PopulationUpdateOptions.UpdateMethod = "WeightedMixing";
trainOpts.PopulationUpdateOptions.InitialStandardDeviation = 0.25;
trainOpts.PopulationUpdateOptions.InitialStandardDeviationBias = 0.25;
